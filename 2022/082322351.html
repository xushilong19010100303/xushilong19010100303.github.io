哈希（散列）

解决的问题

实现数据和存储位置的一一对应。
例如 ：
对一个数据进行一些操作，返回一个数(存储位置)就行。可能会出现冲突。

String类的方法

综述

String(字符串)
他有很多方法，方法都是有括号的，需要先实例化String s = “abCD!&?123”
常见方法

String s = "abCD!&?123"
1 s.charAt(5)              //返回& 
2 s.compareTo("abc")       //返回8,因为比abc长8位，否则返回首位ASCII差,可IgnoreCase!
3 s.contact("QWE")         //拼接到后面 
4 s.contentEquals("abC")   //返回true 括号里必须是Stringbuffer类
5 s.endsWith("123")        //返回true 
6 s.hashCode()             //返回哈希码(ASCII，
7 s.indexOf('1',5)         //第6个位置向后'1'的索引7，也可用ASCII值，还可"123"，还可lastIndexOf
8 s.length()               //返回10 
9 s.replace('a','A')       //a替换为A 
                           //replaceAll(正则，"newstr") replaceFirst
10 s.split("D",100)        //返回的是String[]!!!!! 份数是limit 不够就不处理。
11 s.substring(1,3)        //返回"bCD"
12 s.toLowerCase()         // 转小写，toUpperCase
13 s.valueOf('1'，'s'，'!')//返回1s! 即任意类型转字符串表达形式
14 s.isEmpty()             //false

附：
hashCode方法 $\sum\limits_{i=0}^{n-1}s[i]*31^i$

equals 与 ==

String s1 = "Hello";              // String 直接创建
String s2 = "Hello";              // String 直接创建
String s3 = s1;                   // 相同引用
String s4 = new String("Hello");  // String 对象创建
String s5 = new String("Hello");  // String 对象创建

s1 == s2;         // true, s1 和 s2 都是直接创建
s2 == s3;         // true 
s1 == s4;         // false, 不同引用地址
s4 == s5;         // false, 堆中不同引用地址
 
s1.equals(s3);    // true, 相同内容
s1.equals(s4);    // true, 相同内容
s4.equals(s5);    // true, 相同内容
