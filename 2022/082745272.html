手写instanceof

function instanceOf(child, father) { 
    let R = father.prototype; // 取右表达式的 prototype 值
    L = child.__proto__; // 取左表达式的__proto__值
    while (true) {
        if (L === null) {
            return false;    
        }
        if (L === R) {
            return true;    
        } 
        L = L.__proto__ 
    }}

null 和 undefined 的区别

  • null表示栈中变量没有指向堆中的内存对象
  • undefined不初始化变量、访问对象中未定义属性、函数形参未传入实参

数组和伪数组

  • 伪数组是自定义length属性的对象，但instanceof proto.constructor 都是object
  • Array.from()转换 取决于length 索引不连续时 自动补undefined 索引只考虑0“0” 1 ..无负数等
  • Array.prototype.slice.call()转换

异步相关

回调函数

    setTimeOut(function callback(){},3000)
    setTimeOut(()=>{},3000) //箭头函数简写

执行代码先放在调用栈 setTimeOut是一个API 表示3s后执行 就直接从栈里移除了
3秒以后 （此时同步一定执行完了）Event Loop循环找回调函数

回调地狱 指多个setTimeOut嵌套 可读性变差

Promise

new Promise

  • resolve => then => finally 可加多个.then 避免回调地狱
  • reject => catch => finally
  • 三种状态 pending fulfiled rejected

async await

本质是语法糖

     async function xsl1(){return 4} //自动封装为promise对象返回
     async function xsl2(){return Promise.resolve(4)}
     Promise.resolve(4).then(a=>{函数体}) 
     const a = await Promise.resolve(4) //等价 await 4  /await xsl()

执行顺序问题

  • 先同步，后异步
  • async函数内部的await后面相当于setTimeOut了

宏任务和微任务

  • 宏任务 setTimeOut setInterval DOM事件 AJAX请求
  • 微任务 Promise async/await
    微任务执行早于宏任务

defer 和 async

同步执行 遇到script标签内如果有外部链接网络慢，需要等待很久才继续加载HTML
async 异步执行 下载同时加载页面 但脚本不一定是否在加载完成前执行
defer 推迟 要求脚本必须在页面加载完再执行 适用于与DOM相关

Promise.All

    Promise.all([a,b,c]).then((res)=>{})
  • 表示a,b,c三个promise对象都完成后执行res函数
